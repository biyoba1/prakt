# Алгоритм определения точек пересечения геометрических объектов

## Вариант 15

### Задача
Разработать подпрограммы определения точек пересечения:
1. Прямой и окружности
2. Двух окружностей

---

## 1. Алгоритм нахождения точек пересечения прямой и окружности

### Входные данные:
- Прямая задается двумя точками: (x₁, y₁) и (x₂, y₂)
- Окружность задается центром (cx, cy) и радиусом r

### Выходные данные:
- Список точек пересечения (0, 1 или 2 точки)

### Математическая модель:

Прямая в параметрической форме:
```
P(t) = P₁ + t(P₂ - P₁)
x(t) = x₁ + t(x₂ - x₁)
y(t) = y₁ + t(y₂ - y₁)
```

Уравнение окружности:
```
(x - cx)² + (y - cy)² = r²
```

Подставляя параметрическое уравнение прямой в уравнение окружности, получаем квадратное уравнение:
```
At² + Bt + C = 0

где:
A = (x₂ - x₁)² + (y₂ - y₁)²
B = 2[(x₁ - cx)(x₂ - x₁) + (y₁ - cy)(y₂ - y₁)]
C = (x₁ - cx)² + (y₁ - cy)² - r²
```

Дискриминант:
```
D = B² - 4AC
```

### Анализ решений:

1. **D < 0**: Нет точек пересечения (прямая не пересекает окружность)
2. **D = 0**: Одна точка пересечения (прямая касается окружности)
3. **D > 0**: Две точки пересечения

### Вычисление точек пересечения:

```
t₁ = (-B + √D) / (2A)
t₂ = (-B - √D) / (2A)

x₁ = x₁ + t₁(x₂ - x₁)
y₁ = y₁ + t₁(y₂ - y₁)

x₂ = x₁ + t₂(x₂ - x₁)
y₂ = y₁ + t₂(y₂ - y₁)
```

### Блок-схема:

```
Начало
   ↓
Ввод: x₁, y₁, x₂, y₂, cx, cy, r
   ↓
Проверка: r > 0?
   ↓ (нет) → Ошибка
   ↓ (да)
Проверка: (x₁,y₁) ≠ (x₂,y₂)?
   ↓ (нет) → Ошибка
   ↓ (да)
Вычислить A, B, C
   ↓
Вычислить D = B² - 4AC
   ↓
D < 0? → Вывод: Нет точек
   ↓ (нет)
D = 0? → Вычислить 1 точку → Вывод
   ↓ (нет)
Вычислить 2 точки → Вывод
   ↓
Конец
```

## 2. Алгоритм нахождения точек пересечения двух окружностей

### Входные данные:
- Окружность 1: центр (cx₁, cy₁), радиус r₁
- Окружность 2: центр (cx₂, cy₂), радиус r₂

### Выходные данные:
- Список точек пересечения (0, 1, 2 точки или бесконечность)

### Математическая модель:

Расстояние между центрами:
```
d = √[(cx₂ - cx₁)² + (cy₂ - cy₁)²]
```

### Анализ случаев:

1. **d > r₁ + r₂**: Окружности не пересекаются (слишком далеко)
2. **d < |r₁ - r₂|**: Одна окружность внутри другой (нет пересечений)
3. **d = 0 и r₁ = r₂**: Окружности совпадают (бесконечное количество точек)
4. **d = r₁ + r₂**: Окружности касаются снаружи (одна точка)
5. **d = |r₁ - r₂|**: Окружности касаются изнутри (одна точка)
6. **Иначе**: Две точки пересечения

### Вычисление точек пересечения:

Расстояние от центра первой окружности до линии, соединяющей точки пересечения:
```
a = (r₁² - r₂² + d²) / (2d)
```

Координаты точки на линии центров:
```
px = cx₁ + a(cx₂ - cx₁) / d
py = cy₁ + a(cy₂ - cy₁) / d
```

Высота от линии центров до точек пересечения:
```
h = √(r₁² - a²)
```

Точки пересечения:
```
x₁ = px + h(cy₂ - cy₁) / d
y₁ = py - h(cx₂ - cx₁) / d

x₂ = px - h(cy₂ - cy₁) / d
y₂ = py + h(cx₂ - cx₁) / d
```

### Блок-схема:

```
Начало
   ↓
Ввод: cx₁, cy₁, r₁, cx₂, cy₂, r₂
   ↓
Проверка: r₁ > 0 и r₂ > 0?
   ↓ (нет) → Ошибка
   ↓ (да)
Вычислить d
   ↓
d = 0 и r₁ = r₂? → Исключение: окружности совпадают
   ↓ (нет)
d > r₁ + r₂? → Вывод: Нет точек
   ↓ (нет)
d < |r₁ - r₂|? → Вывод: Нет точек
   ↓ (нет)
Вычислить a, px, py
   ↓
Вычислить h² = r₁² - a²
   ↓
h² ≈ 0? → Вычислить 1 точку → Вывод
   ↓ (нет)
Вычислить 2 точки → Вывод
   ↓
Конец
```

## 3. Классы эквивалентности для тестирования

### Для пересечения прямой и окружности:

| № | Класс эквивалентности | Описание |
|---|----------------------|----------|
| 1 | Две точки пересечения | Прямая пересекает окружность |
| 2 | Одна точка (касание) | Прямая касается окружности |
| 3 | Нет точек | Прямая не пересекает окружность |
| 4 | Недопустимые данные | r ≤ 0 |
| 5 | Недопустимые данные | Совпадающие точки прямой |

### Для пересечения двух окружностей:

| № | Класс эквивалентности | Описание |
|---|----------------------|----------|
| 1 | Две точки пересечения | Окружности пересекаются |
| 2 | Одна точка (внешнее касание) | d = r₁ + r₂ |
| 3 | Одна точка (внутреннее касание) | d = \|r₁ - r₂\| |
| 4 | Нет точек (далеко) | d > r₁ + r₂ |
| 5 | Нет точек (одна внутри) | d < \|r₁ - r₂\| |
| 6 | Бесконечность точек | Окружности совпадают |
| 7 | Недопустимые данные | r₁ ≤ 0 или r₂ ≤ 0 |

## 4. Особенности реализации

### Обработка погрешностей вычислений:
Используется константа EPSILON = 10⁻¹⁰ для сравнения вещественных чисел:
```csharp
const double EPSILON = 1e-10;

if (Math.Abs(value) < EPSILON)
{
    // Считаем значение равным нулю
}
```

### Обработка исключительных ситуаций:
1. **ArgumentException**: Некорректные входные данные (отрицательный радиус, совпадающие точки)
2. **InvalidOperationException**: Особые случаи (совпадающие окружности)

### Возвращаемое значение:
Методы возвращают `List<Point>`, где:
- Пустой список — нет пересечений
- 1 элемент — касание
- 2 элемента — две точки пересечения

## 5. Пример вычислений

### Пример 1: Прямая и окружность (2 точки)

**Входные данные:**
- Прямая: от (-5, 0) до (5, 0)
- Окружность: центр (0, 0), радиус 3

**Вычисления:**
- A = (5-(-5))² + (0-0)² = 100
- B = 2[(-5-0)(10) + (0-0)(0)] = -100
- C = (-5)² + 0² - 9 = 16
- D = 10000 - 6400 = 3600
- t₁ = (100 + 60)/200 = 0.8 → (3, 0)
- t₂ = (100 - 60)/200 = 0.2 → (-3, 0)

**Результат:** Точки (3, 0) и (-3, 0)

---

### Пример 2: Две окружности (2 точки)

**Входные данные:**
- Окружность 1: центр (0, 0), радиус 3
- Окружность 2: центр (4, 0), радиус 3

**Вычисления:**
- d = 4
- a = (9 - 9 + 16) / 8 = 2
- px = 0 + 2 = 2
- py = 0
- h² = 9 - 4 = 5
- h = √5 ≈ 2.236
- Точка 1: (2, 2.236)
- Точка 2: (2, -2.236)

**Результат:** Точки (2.0, 2.236) и (2.0, -2.236)

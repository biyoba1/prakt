## 3. Математическая формулировка задачи

### 3.1 Описание входных данных

**Для пересечения прямой и окружности:**
- x₁, y₁ — координаты первой точки прямой (вещественные числа)
- x₂, y₂ — координаты второй точки прямой (вещественные числа)
- cx, cy — координаты центра окружности (вещественные числа)
- r — радиус окружности (вещественное число, r > 0)

**Для пересечения двух окружностей:**
- cx₁, cy₁ — координаты центра первой окружности (вещественные числа)
- r₁ — радиус первой окружности (вещественное число, r₁ > 0)
- cx₂, cy₂ — координаты центра второй окружности (вещественные числа)
- r₂ — радиус второй окружности (вещественное число, r₂ > 0)

### 3.2 Описание выходных данных
- Количество точек пересечения (0, 1 или 2)
- Координаты точек пересечения (если они есть)
- Сообщения о специальных случаях (совпадение окружностей, отсутствие пересечений)

### 3.3 Функциональные характеристики программы
Программа должна:
1. Предоставлять выбор между двумя режимами работы
2. Вычислять точки пересечения прямой и окружности
3. Вычислять точки пересечения двух окружностей
4. Обрабатывать все возможные случаи (0, 1, 2 точки пересечения)
5. Генерировать исключения при некорректных входных данных
6. Проходить юнит-тесты по всем классам эквивалентности

## 4. Структура классов

### Класс Point
Представляет точку на плоскости:
- Свойства: X, Y (координаты)
- Метод ToString() для форматированного вывода

### Класс GeometryHelper
Содержит статические методы для геометрических вычислений:

**1. FindLineCircleIntersection()**
- Параметры: координаты двух точек прямой, центр и радиус окружности
- Возвращает: List<Point> с точками пересечения
- Генерирует исключения при некорректных данных

**2. FindCircleCircleIntersection()**
- Параметры: центры и радиусы двух окружностей
- Возвращает: List<Point> с точками пересечения
- Генерирует исключения при некорректных данных и совпадении окружностей

**3. ValidateCoordinates()**
- Вспомогательный метод проверки валидности координат

Получить практические навыки разработки графических приложений с использованием таймера, обработки событий клавиатуры и мыши, создания второй формы для настройки параметров.

## 3. Математическая формулировка задачи

### 3.1 Описание входных данных
- Размер квадрата: 40 пикселей
- Начальная позиция: (10, 10)
- Начальный цвет: красный
- Начальное направление: вправо
- Скорость движения: настраиваемая (интервал таймера 20-180 мс)
- Цвет квадрата: настраиваемый через ColorDialog

### 3.2 Описание выходных данных
- Анимация движущегося квадрата
- Визуальное отображение смены цвета при изменении направления
- Сообщение при остановке по нажатию клавиши

### 3.3 Функциональные характеристики программы
Программа должна:
1. Обеспечивать движение квадрата по периметру формы (не проходя нижнюю границу)
2. Двигать квадрат в прямом и обратном направлении по периметру
3. Менять цвет квадрата при каждом изменении направления движения
4. Останавливаться при нажатии любой клавиши
5. Адаптироваться к изменению размеров формы
6. Предоставлять настройку цвета и скорости через вторую форму

## 4. Структура проекта

### Form1 (Главная форма)
**Переменные состояния:**
- `squareX`, `squareY` — координаты квадрата
- `squareSize` — размер квадрата (40 пикселей)
- `squareColor` — текущий цвет
- `direction` — направление (0-3)
- `speed` — скорость движения

**Методы:**
- `InitializeSquare()` — инициализация начальных значений

- `ChangeColor()` — изменение цвета при смене направления
- `OnPaint()` — отрисовка квадрата
- `timer1_Tick()` — обработчик таймера
- `button1_Click()` — запуск/остановка
- `button2_Click()` — открытие настроек
- `Form1_KeyPress()` — обработка нажатия клавиши
- `Form1_Resize()` — адаптация к изменению размера

### Form2 (Форма настроек)
**Свойства:**
- `SquareColor` — выбранный цвет
- `Speed` — выбранная скорость

**Методы:**
- `button1_Click()` — выбор цвета через ColorDialog
- `trackBar1_Scroll()` — изменение скорости
- `UpdateColorDisplay()` — обновление отображения цвета
- `UpdateSpeedDisplay()` — обновление отображения скорости

## 8. Логика движения

### Направления движения:
0. **Вправо** (→) - цвет: Красный (Red)
1. **Вниз** (↓) - цвет: Синий (Blue)
2. **Влево** (←) - цвет: Зеленый (Green)
3. **Вверх** (↑) - цвет: Оранжевый (Orange)

### Траектория:

```
(10,10) ────────────────► (maxX, 10)
  ▲         RED                │
  │                      BLUE  │
  │                            ▼
  │                      (maxX, maxY)
  │                            │
ORANGE                  GREEN  │
  │                            ▼
(10, maxY) ◄──────────── (10, bottomLimit)

[Нижняя граница формы не используется]
```

### Граничные условия:
- Правая граница: `squareX + squareSize >= ClientWidth - margin`
- Нижняя граница: `squareY + squareSize >= bottomLimit - squareSize`
- Левая граница: `squareX <= margin`
- Верхняя граница: `squareY <= margin`
